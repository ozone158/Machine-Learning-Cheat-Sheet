# MLcheatsheet

Obviously thanks to vibe coding, learning to write good pseudo code is way more important than learning to write actual code, so I made this for referencing and learning. 

## Cheat Sheets

| Paradigm | File | Models / algorithms |
|----------|------|---------------------|
| **Supervised** | [supervised-learning.md](./supervised-learning.md) | Linear & Logistic Regression, Decision Tree, Random Forest, SVM, KNN, Naive Bayes, Neural Network, Gradient Boosting |

## Shared Layout

Every cheat sheet follows the same structure:

| Section | Contents |
|---------|----------|
| **0. Quick Reference** | Comparison tables, modality guides, decision flow |
| **1. Life Cycle** | End-to-end workflow (data → train → evaluate) and metrics overview |
| **2. Models / Architectures** | One subsection per model with foldable **a–g** blocks |
| **3. Dictionary** | A–Z term index with cross-links |

### Foldable blocks (a–g)

Each model in Section 2 includes:

- **a.** Scenarios & cons
- **b.** Notations & dimensions
- **c.** Math core (line-by-line LaTeX)
- **d.** Update rules
- **e.** Hyperparameters & tuning
- **f.** Metrics
- **g.** Example (pseudocode)
